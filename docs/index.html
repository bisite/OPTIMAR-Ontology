Ontology Specification Draft

Abstract

Ontology modeling structural elements, sensors, events, and failures in floating offshore wind platforms, including measurement data, involved actors, and semantic relationships between components. Compatible with SOSA for observations and QUDT for units of measure.

Offshore Wind Power Platform Ontology for OPTiMAR: Description back to ToC

Version 2.0 (2025-12-02) - Major improvements: - Added optimar:Generator class for turbine generators - Added inverse properties: optimar:isHousedIn, optimar:detects, optimar:affects - Declared optimar:connectedTo as symmetric property - Renamed properties to avoid class name collisions: optimar:deformationLevel, optimar:misalignmentLevel - Corrected optimar:affectedBy domain to optimar:Component (previously optimar:TurbineComponent) - Added optimar:StructuralComponent as subclass of optimar:Component - Added structural restrictions to TurbineComponent classes - Separated failure classes from failure event classes - Fixed restriction syntax in optimar:Maintenance and optimar:Observation - Improved consistency in QUDT usage for physical quantities - Added cardinality constraints to optimar:Turbine class
